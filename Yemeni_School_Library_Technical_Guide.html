أولاً: الفلسفة الأكاديمية ورؤية بوابة المناهج اليمنية

في ظل الظروف التعليمية الراهنة التي يمر بها الطلاب والكوادر التربوية في الجمهورية اليمنية، يبرز غياب الكتاب المدرسي الورقي كأحد أكبر التحديات المعرقلة للتحصيل المعرفي. ومن هذا المنطلق الأكاديمي، تأسست المكتبة المدرسية اليمنية الشاملة كذراع معرفي أساسي بداخل "منصة اكتفاء التعليمية".

الركائز الست في فلسفة النظام البرمجي للمكتبة:
  • الشمولية والفرز: تغطية الإثني عشر صفاً دراسياً كاملاً (من الأول الابتدائي إلى الثالث الثانوي بقسميه العلمي والأدبي) وتعبئتها تزامناً مع الفهرس الوزاري المعتمد.
  • الاستدامة والعمل دون اتصال (Offline-First): توفير بنية تحتية برمجية تدعم المذاكرة دون الحاجة الدائمة للاتصال بالإنترنت من خلال دمج الأغطية الافتراضية والكتب المحلية.
  • محاكاة الواقع الحسي (High Fidelity Sensory): الانتقال من العرض التقليدي الجامد للمستندات إلى بيئة تفاعلية خشبية مريحة تحاكي بيئات التعلم الطبيعية وتقلل من إرهاق العين.
  • تكامل الوسائط السمعية والبصرية: إلحاق قارئ نصوص فوري ذكي (Text-to-Speech Engine) باللغة العربية لمساعدة ذوي الاحتياجات وبراعم المراحل الابتدائية.
  • التكيف اللوحي والخلوي (Adaptive Grid): واصفات تصميم مرنة تعكف على تشغيل التطبيق بنسق مذهل على الشاشات المنزلية والهواتف واللوحيات والأجهزة القابلة للطي بمثالية.
  • آليات الربط المفتوح والتبادل: دعم تحميل المناهج من خوادم شبكة تلغرام المفتوحة مع إمكانيات فرز واستيراد المستندات المحلية من الهاتف وبطاقات SD الخارجية بسلاسة وذكاء.

ثانياً: معمارية وهندسة التطبيق ومتطلبات البناء

يعتمد التطبيق على معمارية MVVM (Model-View-ViewModel) لضمان الفصل الكامل بين منطق البيانات وطبقة التصميم، وخضع البناء لأطر التحديث المتقن للتوافق المطلق مع شفرات التجميع الحديثة.

المخطط الهندسي لتدفقات البيانات السحابية والتخزين المحلي بداخل المكتبة:

+-----------------------------------------------------------------------------------------+ | طبقة العرض والواجهة السينمائية | | (SchoolLibraryActivity -> Entry Portal -> Shared State View Engine) | +-----------------------------------------------------------------------------------------+ | (طلب تصفح / تصفية / بحث / تحميل) ^ (حالات الواجهة المحدثة بالملي ثانية) v | +-----------------------------------------------------------------------------------------+ | طبقة معالجة البيانات والمنطق | | (ViewModel Logic -> Coroutines Dispatchers -> TTS -> File Decoders) | +-----------------------------------------------------------------------------------------+ | (عمليات قاعدة البيانات) ^ (تزامن فوري) v | +----------------------------------------------------------+------------------------------+ | | | | +----------------------------------------------+ | +--------------------+ | | | قاعدة بيانات Room المحلية | | | عارض الكتب والوسائط| | | | (SchoolBookEntity <=> SchoolBookDao/Database)| | | (Interactive PDF) | | | +----------------------------------------------+ | +--------------------+ | | | | | | +--------------------------+-------------------------------+--------------+---------------+ | (قنوات التحقق السلوكي والحفظ النهائي) v +----------------------------------------------+ | نظام الملفات للآندرويد | | (/app/files/books/ & /covers/ ... ) | +----------------------------------------------+

تفصيل كتالوج المجلدات والملفات ووظيفتها البنائية في مشروع المكتبة:

مسار واجهة الملف الوصف الفني للمكون ودوره في المشروع

رابعاً: محرك الرفوف الخشبية والتأثيرات الحركية والسينمائية للبوابة

تحقيقاً لمعايير التصميم الرفيعة وخروجاً عن الأنماط المألوفة، تم ابتكار واجهة الرفوف الخشبية الذكية (Sleek Wooden Shelves View) التي تقسم كراسات ومواد التعليم حسب الرفوف الحقيقية للمكتبات، مُطعمة بتأثيرات حركية سينمائية تضفي حياة ولمسات جمالية أثناء التنقل بمساعدة مكتبة Jetpack Compose Animation.

الخصائص التصميمية والتأثيرات السينمائية المضمنة:
  • تأثير الفرز السينمائي للخلفيات (Background Shimmer & Blur): عند فتح المكتبة يستقبلك التطبيق ببوابة سينمائية مكونة من مصراعين ذهبيين يفتحان تدريجياً لليمين واليسار مع تلاشي الضبابية لتهيئة عين الطالب بصرياً لبدء مسار التعلم.
  • حواف الرفوف الخشبية ثلاثية الأبعاد (3D Wooden Shelving Effect): يتم تمثيل الرفوف بتدرجات دائرية خشبية غنية #D7CCC8 → #BCAAA4 → #A1887F بالإضافة لحاشيات سميكة غامقة تحت محاذاة الكتب لرسم تأثير العمق الظلي.
  • ديناميكيات الارتداد والتفاعل (Cinematic Spring Physics): تتوفر بطاقات الكتب على تموج مشع ذهبي عند الضغط عليها، وتتكيف أحجام الغلاف تلقائياً مع ارتدادات فيزيائية لطيفة (Scale Transitions) لإدراك لمس العنصر.
  • التلاؤم اللوحي الكامل: تم التحديث والانتقال من تخطيط الصفوف البسيطة إلى تخطيط مصفوفي متكيف لضمان توزيع الكتب في صفوف واسعة ومنظمة داخل أجهزة الـ iPad والأجهزة اللوحية العريضة.

خامساً: القارئ التفاعلي المتقدم المدمج ومحرك النطق الصوتي الفوري للبث

لا يعتمد التطبيق على تطبيقات خارجية لعرض المستندات، بل يشتمل بداخله على قارئ PDF تفاعلي أصلي ذو ميزات متقدمة تهدف لربط الطالب بالصفحات بشكل حسي، بالإضافة لمحرك تلاوة ونطق النصوص فائق الجودة لدعم الطالب.

تتم تهيئة القارئ التفاعلي بالملي ثانية ليحتفظ بقنوات الاستخدام والتشغيل للعمليات التالية دون خلل:

📖 التصفح بنمط الصحف التقليدية

دعم الإزاحة والتمرير العمودي والأفقي اللانهائي للصفحات بلمسات اليد الفورية، مع تذكر كامل وفوري لآخر صفحة تم تصفحها وإعادتها فور الدخول الثاني للكتاب.

🌙 الوضع الليلي التفاعلي المريح

تبديل شامل لرموز الإضاءة وتحويل النص والخلفيات بداخل الكتب إلى اللون الرمادي والأسود الخافض لتوهج أشعة الشاشات وحماية بصر القارئ ليلاً.

🔊 القارئ الصوتي الناطق للنص (TTS)

دمج فوري وبسيط لـ TextToSpeech API باللغة العربية لنطق فقرات الكتاب والشرح بمجرد تمرير الإصبع أو الضغط على زر استماع، مع تحكم فوري بالبدء والإيقاف المؤقت.

✍️ التعليقات والملاحظات الهامشية

إمكانية تدوين ملاحظات تفاعلية بجانب مواضيع الدروس لمراجعتها في أوقات الامتحانات بملفات نصية معزولة مربوطة بهوية الكتاب محلياً.

سادساً: نظام البحث الموحد والمزامنة السحابية وقنوات الاستيراد المحلية

لتبسيط تجربة التصفح، يحتوي التطبيق على محرك بحث فوري يفحص العناوين والمواد والصفوف بشكل ديناميكي، بالإضافة إلى خيارات استيراد الملفات من الذاكرة المحلية والشبكة وتوطيد صلتها بالفهرس المعتمد.

أبعاد الربط والاستيراد المتقدمة:
  • البحث النصي المعزز بمحو التشكيل والرموز (Arabic Normalization): يبحث المحرك في قواعد البيانات بطريقة متقدمة تسمح بإهمال همزات الألف ومقاطع التاء المربوطة والهاء والشد لتسريع الحصول على الكتب بمجرد بضع ملامسات للأزرار.
  • جسر الربط والمزامنة التلقائية (Index Matching Resolver): عند اختيار الطالب لاستيراد ملف PDF محلي خارجي، يقوم المعالج بمقارنة الاسم المدخل تلقائياً مع العناوين والصفوف بداخل الفهرس، وإذا وجد تطابقاً، يقوم بربط الملف المحلي المذكور فوراً ببيانات الكتاب في قاعدة البيانات وتحديث زره إلى اللون الأخضر.
  • خادم تكرار وإعمار افتراضي للكتب (Offline Fallback Simulator): في حال غياب الاتصال بالشبكة لروابط التيليجرام أو عدم توفر الملف، يستجيب النظام بشكل ذكي عبر كتابة كراسات تعليمية فورية قصيرة محلياً تمنح الطالب مادة المذاكرة السريعة لمنع توقف عملية التصفح والامتحانات لديه.

سابعاً: سجل المعارك التقنية وحلول عثرات البناء والترجمة البرمجية

خلال رحلة التطوير وإعداد وبناء بوابة المناهج اليمنية، واجهنا بعض التحديات البرمجية وتم التغلب عليها وضبطها لتفادي حدوث مثلها في الأجهزة مستقبلاً:

🛑 العقبة الأولى: فشل الترجمة البرمجية بسبب 'tabIndicatorOffset'

سبب العلة: حاولت الشفرات الأولية استخدام دالة مؤشر تبويبات متطور لا يتضمن الدعم من حزم التصميمات الملحقة في النظام، مما أدى لخلل في مجمع الكود compileDebugKotlin وتوقف البناء تماماً.

✅ الحل والعلاج البرمجي المطبق:

قمنا بقشط وتبسيط تخصيص أسلوب المؤشر (tabIndicatorOffset) والاعتماد على السلوك والمؤشر الافتراضي لـ TabRow بخصائص لونية مخصصة، مما استعاد سرعة واستتباب التجميع الفوري وتجاوز الخلل للأبد.

🛑 العقبة الثانية: تقطع واجهات Grid الخلوية في الشاشات اللوحية الكبيرة والأيباد

سبب العلة: كان التطبيق يستهدف إسناد أعمدة مصفوفية ثابتة في الهواتف العادية GridCells.Fixed(2/3)، مما تسبب في ظهور فجوات وتمددات مسيئة من الناحية الجمالية للكتب على شاشات اللوحيات.

✅ الحل والعلاج البرمجي المطبق:

أجرينا ترقية تصميمية شاملة للاعتماد على GridCells.Adaptive(minSize = 150.dp) للمربعات الهامة للكتب والخيارات، مما يدعم تزايد الصفوف واستعراض الكتب بجوار بعضها البعض بشكل متناسق يليق بمشروع اكتفاء الأكاديمي.
